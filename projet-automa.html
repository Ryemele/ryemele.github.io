Résumé

Summary

Plateforme robotique inspirée véhicule connecté : deux TurtleBot3 orchestrés par ROS, communiquant via MQTT chiffré en TLS/SSL et ITS-G5 pour la V2I/V2V. Données LiDAR, caméra, batterie et vitesse transmises de façon sécurisée vers un broker VPS et vers des pairs, puis visualisées sur une interface React/TypeScript.

Robotics platform inspired by connected-vehicle use cases: two TurtleBot3 controlled with ROS, exchanging data over TLS/SSL-encrypted MQTT and ITS-G5 for V2I/V2V. LiDAR, camera, battery and velocity data securely published to a VPS broker and peers, visualized in a React/TypeScript interface.

Stack & architecture

Stack & architecture

  • ROS Noetic (Ubuntu 20.04) : nodes, topics, launch, TF, /scan, /cmd_vel, /odom.
  • Broker MQTT (Mosquitto) sur VPS : V2I + V2V médié (publish/subscribe, reconnexion).
  • Bridge ROS→MQTT (Python) : capteurs vers JSON (LiDAR, caméra, batterie, vitesse).
  • Web UI React/TypeScript : contrôle, carte capteurs, diagnostics via MQTT/WebSockets/ROSBridge.
  • Naming dynamique par IP : robotX/* (e.g. robot7/scan), évite conflits inter-robots.

Défis techniques & décisions clés

Technical challenges & key decisions

  • Synchronisation capteurs/commande : tuning du loop rate + QoS sur les topics critiques pour garantir la réactivité sans à-coups.
  • ITS-G5 opérationnel : la stack ITS-G5 a été intégrée pour permettre une communication directe V2V en parallèle du broker MQTT, réduisant la latence et respectant les standards ITS européens.
  • TLS/SSL (MQTTs) : toutes les communications avec le broker VPS sont chiffrées en TLS/SSL (port 8883), avec certificats X.509 pour authentification et intégrité des flux (LiDAR, caméra, odométrie).
  • Scalabilité : schéma de noms dynamiques par IP (robotX/) garantissant l’unicité des topics, le debug et l’ajout de robots sans reconfiguration manuelle.
  • Infra données : API+DB pour stocker la télémétrie en temps réel (positions, alertes, diagnostics) et la visualiser depuis le dashboard web React/TS.
  • Sensor/command sync: tuned loop rate + QoS on critical topics to ensure responsive control without halts.
  • ITS-G5 implemented: the ITS-G5 stack was integrated to provide direct V2V communication alongside the MQTT broker, reducing latency and complying with European ITS standards.
  • TLS/SSL (MQTTs): all broker communications are encrypted with TLS/SSL (port 8883), using X.509 certificates for authentication and ensuring data integrity (LiDAR, camera, odometry streams).
  • Scalability: dynamic IP-based naming scheme (robotX/) ensures topic uniqueness, clean debugging, and seamless fleet growth.
  • Data infra: API+DB collect telemetry in real time (position, alerts, diagnostics) and feed the React/TS dashboard for visualization.
