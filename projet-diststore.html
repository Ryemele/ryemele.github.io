Résumé

Summary

Conception et développement d’un mini Dropbox/IPFS sécurisé : chaque fichier est découpé, chiffré côté client (AES-256 + PBKDF2), puis distribué sur plusieurs nœuds de stockage. Le système assure la tolérance aux pannes et la confidentialité de bout en bout.

Design and implementation of a secure mini Dropbox/IPFS: each file is chunked, encrypted client-side (AES-256 + PBKDF2), then distributed across multiple storage nodes. The system ensures fault tolerance and end-to-end confidentiality.

Objectifs

Objectives

  • Implémenter un service central gérant les chunks, la réplication et les métadonnées.
  • Développer une API REST/gRPC fiable pour PUT/GET/HEAD sur les chunks.
  • Assurer l’intégrité et la confidentialité via chiffrement côté client et hash SHA-256.
  • Gérer la tolérance aux pannes (perte d’un nœud sans perte de fichier).
  • Implement a central service managing chunks, replication, and metadata.
  • Develop a reliable REST/gRPC API for PUT/GET/HEAD chunk operations.
  • Ensure data integrity and confidentiality using client-side AES encryption + SHA-256 hashing.
  • Handle fault tolerance (node failure recovery without file loss).

Architecture logicielle

Software architecture

  • Interface Web (Flask/React) : upload/download, chiffrement côté client.
  • Service central : découpe, réplication, mapping des chunks/nœuds.
  • Nœuds de stockage : API PUT/GET/HEAD, persistance locale.
  • Base de métadonnées : structure, propriétaires, clés chiffrées.

Résultats & démonstration

Results & demonstration

  • Upload/download complet via interface web sécurisée TLS/SSL.
  • Résilience validée : fichier lisible après arrêt d’un nœud.
  • Partage sécurisé entre utilisateurs (échange de clés chiffrées).
  • Full upload/download through a TLS/SSL-secured web interface.
  • Resilience validated: file accessible after one node failure.
  • Secure sharing between users (encrypted key exchange).
