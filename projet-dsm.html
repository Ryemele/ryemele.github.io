Résumé

Summary

Conception et implémentation complète d’une mémoire partagée distribuée (DSM) permettant à plusieurs processus répartis sur différentes machines d’accéder à une même plage d’adresses mémoire virtuelle. Le système capture les erreurs de segmentation (SIGSEGV) et transfère dynamiquement la propriété des pages entre processus via des sockets TCP. Le projet s’est déroulé sur 2 phases : dsmexec (lancement & communication des processus) et bibliothèque DSM (gestion mémoire et cohérence).

Design and implementation of a Distributed Shared Memory (DSM) system allowing multiple processes across different machines to access a common virtual memory space. The system intercepts segmentation faults (SIGSEGV) and dynamically transfers page ownership between processes via TCP sockets. The project was structured in two phases: dsmexec (process launching & communication) and DSM library (memory management & consistency).

Architecture & Techniques

Architecture & Techniques

  • dsmexec : lance les processus distants via SSH et gère les connexions TCP.
  • dsmwrap : wrapper intermédiaire pour initialiser les variables d’environnement.
  • dsm_init : connexion entre pairs et allocation des pages en round-robin.
  • segv_handler : rattrape SIGSEGV et déclenche les requêtes DSM_REQ / DSM_PAGE.
  • dsm_comm_daemon : thread d’écoute permanent pour traiter les requêtes.
  • dsmexec: launches remote processes via SSH and manages TCP connections.
  • dsmwrap: intermediate wrapper initializing environment variables.
  • dsm_init: establishes peer connections and allocates pages round-robin.
  • segv_handler: catches SIGSEGV and triggers DSM_REQ / DSM_PAGE messages.
  • dsm_comm_daemon: background thread handling incoming page requests.

Protocoles & Messages

Protocols & Messages

  • DSM_REQ : demande d’accès à une page manquante.
  • DSM_PAGE : transfert d’une page au processus demandeur.
  • DSM_NEWPROP : notification de changement de propriétaire de page.
  • DSM_FINALIZE : synchronisation et terminaison propre des processus.
  • DSM_REQ: request for a missing memory page.
  • DSM_PAGE: transfer of a memory page to the requesting process.
  • DSM_NEWPROP: broadcast page ownership update to all peers.
  • DSM_FINALIZE: coordinated clean shutdown of all processes.
